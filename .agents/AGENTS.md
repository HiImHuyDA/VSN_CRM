# CSR Web Developer Guidelines & Project Rules

Welcome, Agent! This file contains essential architecture rules, environment configurations, and business logic requirements for the CSR Web (Customer Service Request) workspace. Please adhere to these strictly.

---

## 🚀 1. Technology & Environments
- **Backend Port**: Running on port `3002` (configured in `backend/.env` under `PORT=3002`).
- **Frontend Port**: Vite dev/preview running on `4173`.
- **Proxy Configuration**: Full routing of `/api` requests to backend `http://localhost:3002` is configured in [vite.config.js](file:///D:/PM%20-%20PD%20Rev/project/csr-web/frontend/vite.config.js).
- **Process Management**: Processes are managed via PM2 (`ecosystem.config.js`). Use `pm2 restart all` to restart, or `pm2 delete all && pm2 start ecosystem.config.js` to clear environment caches.

---

## 💾 2. Database & Stored Procedure Rules
- **No Inline SQL**: Express routes in `backend/src/routes/` MUST NOT execute raw SQL queries inline. All operations must call Stored Procedures via `execute('usp_...')`.
- **Renamed & Standardized Procedures**: The old legacy SP names have been replaced with a standardized naming convention: `usp_[Module]_[Action]_[Scope]`. Use these names in code:
  - `usp_TaskConfig_GetDefaultsByDestinations` (loads task defaults for location list)
  - `usp_TaskConfig_List` (loads list of configs for system config admin UI)
  - `usp_Submission_GetDetail` (loads detailed view of a submission)
  - `usp_Submission_List` (loads paginated and filtered dashboard lists)
  - `usp_EmailTemplate_GetById` (gets single campaign email template)
  - `usp_EmailTemplate_List` (gets email campaign templates list)
  - `usp_TaskAttachment_GetById` (gets attachment by ID)
  - `usp_TaskAttachment_ListByTaskId` (lists task attachments)
  - `usp_Dashboard_GetStats` (generates dashboard charts & counter statistics)
- **Database Migrations**: All database schema changes should be saved in `backend/database/migrations/` and run using `node run_migration.js <migration_file.sql>`.

## 📊 3. SharePoint Excel Synchronization Workflow
- **Integration**: Sync is performed via the Microsoft Graph Excel Table API (`/workbook/tables/{table-name}/rows`).
- **Config Variables**: Defined in `backend/.env` under:
  - `SHAREPOINT_EXCEL_TABLE=Table1`
  - `SHAREPOINT_EXCEL_SHEET=Sheet1`
- **Matching Keys**: When a submission is edited, look up the existing row in SharePoint Excel matching:
  - `CSR_Id` + `ĐỊA ĐIỂM CÔNG TÁC` + `NGÀY ĐI`.
- **Preserve Vehicle Details**: If a matching row is found during edit, preserve any existing driver name, phone number, vehicle info, and start time columns. Do not overwrite them with blank data.
- **Attendee Calculation**:
  - `Total Attendees` = Count of staff in `Attendees` + `GuestCount` (number of guests).
- **Column Mappings**: Ensure correct index alignment when writing the 33 columns to the SharePoint Excel table row.

## ✉️ 4. Email Notifications
- **Trigger**: Sent automatically when BOD approves the hospitality request (`BOD đã duyệt`).
- **Template**: Formatted as a HTML table with mission details (Departure, Return, Destination, Attendees, Vehicle Info, Submitter).
- **Edit Workflow**: If a sheet is edited, check if schedule, destination, or date has changed. If so, resend the notification with the prefix `<div style="color: #d9381e;">Đơn tiếp khách có thay đổi như sau:</div>` above the details table.
- **Recipients**: Active assignees and supervisors of the task, except for Airport Shuttle at VSN-NT.
